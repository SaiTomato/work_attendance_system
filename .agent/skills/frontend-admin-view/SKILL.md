## Views
- 今日出席概览
- 异常员工列表
- 单员工历史记录

## Rules
- 默认只展示“异常”数据以减少管理员认知负荷
- 表格优先于表单（Admin 更倾向于批量查看和快速筛选）
- 所有修改数据的操作必须触发二次确认 (Confirm Dialog)

## Data Rules
- 前端严禁编写出席状态的计算逻辑
- 所有 status 枚举值及颜色映射应根据后端返回的 key 进行映射
- 所有的日期选择器应限制只能选择有效的考勤周期

## When to Use
- 新增管理后台 React 组件或页面
- 修改前端交互
